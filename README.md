## The Problem

In 2025–2026, billions of dollars were lost due to hacks, scams, market crashes, and human error in crypto.

People want:
- **Automation** — they don’t want to manually manage yields and transfers
- **Security** — protection from hacks and emotional mistakes
- **Simplicity** — no seed phrases, no gas fees, no complex DeFi jargon

Existing solutions are either:
- Too manual and stressful
- Fully transparent (easy to front-run or track)
- Not truly autonomous or safe for the coming wave of AI agents

**HALO solves this** by acting as your invisible, always-on **Autonomous Private Guardian**.

---

## The Solution

HALO is a premium **web dashboard** that lets users turn on "Autonomous Protection" with a single toggle.

Once enabled:
- HALO automatically moves idle funds into safe yield opportunities on Flow
- Monitors for market threats and volatility
- Takes protective actions using intelligent agents
- Maintains full transparency for the user via clean logs
- Ensures human oversight and safety at all times

It feels like a high-end security SaaS (think Cloudflare + Apple Security), not a crypto dApp.

### Core User Flow
1. Sign up with **email or passkey** (no wallet needed)
2. Toggle **"Enable Autonomous Protection"**
3. Relax — HALO works 24/7 in the background

## Uniqueness

While most projects build trading bots or simple yield tools, HALO builds a **security primitive**:

- **Simple outside** — One beautiful toggle
- **Sophisticated inside** — Multiple autonomous agents working together safely
- **Human-first** — Always keeps the user in control with clear oversight
- **Agent-native** — Built using ERC-8004 so agents are verifiable economic entities

This makes HALO not just another DeFi app, but a foundational piece for safe agentic finance.

---

## Architecture Highlights

- **Frontend**: Next.js + Tailwind CSS (dark premium cybersecurity aesthetic)
- **Agents**: ERC-8004 registered autonomous guardians with structured `agent.json` manifest and `agent_log.json` execution logs
- **Blockchain**: Flow (Cadence) for all financial actions
- **Storage**: Storacha for active memory + Filecoin for immutable long-term logs
- **Safety**: World ID periodic verification + manual kill-switch

All agent actions are logged, verifiable, and reversible by the human owner.
